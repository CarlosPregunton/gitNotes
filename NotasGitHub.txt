git init 				// Convert directory into a repository directory
git clone 				// Clone an entire remote repository
git clone -b <branch name> <SSH>	// Clone a specific branch from a remote repository

git add	<file.py>			// Add an specified file in working directory to staging area.
git add .				// Add all modified files in working directory to staging area.
git commit -m "commit here"		// Take a in-staging-area files snapshot.
git status
git log					//List commit history

git branch				// List all branches
git branch "new branch name"		// Create a new branch.
git branch -D <branch name>		// Deleate a branch
git checkout <branch name>		// Change from branch to another.
git checkout -b <new branch name>	// Creates a new branch and checkout to it.

git commit --ammend -m "new_message"	// Correct commit's message
git reverse <sha>			// Undo a specified commit
git merge master -m "commit message"	//Get latest changes from master when in no master branch.

git push origin <branch_name>		// Push changes to remote repository.
git pull 				// Get latest changes from master branch.


What does --, -, -a, -v, -r, -l, -m means in git commands?
 
-b: operate over a specific branch.
-a: stands for all.
-l: stands for list.
-m: to specify a message.
-v: stands for verbose (give more detail).
